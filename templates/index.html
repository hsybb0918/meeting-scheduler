Multi-Agent Meeting Scheduler

Negotiation Strategies

Three negotiation strategies are implemented in the web framework.

Strategy One

Simple Match

The host agent sends the meeting request to all guest agents. When a guest agent receives a meeting request, it will access the calendar and find if the requested time can be accepted. Each guest agent replies to the host agent whether it is available. After the host agent receives all the replies, if all guest agents are available at the requested time, the host agent informs that the meeting is successfully scheduled, otherwise the counterproposal strategy will be applied.

Strategy Two

Counterproposal

If participants cannot reach an agreement on the requested time, they will start the counterproposal strategy. Once the message is received by a guest agent, it creates a counterproposal and replies with five available time slots that have the highest preference and are closest to the initial requested time. After the host agent receives all the replies, if there is an intersection, the host agent informs that the meeting is successfully scheduled, otherwise the voting strategy will be applied.

Strategy Three

Voting

The voting strategy is the last resort for the agents to find the meeting time when other strategies fail. The host agent finds ten available time slots on that day and sends to the guest agents. Each guest agent votes on each time, informing the host agent the availability and preference of that time. Once all the guest agents respond to the host agent, the host agent picks the time when all participants are available and the total preference is the highest, otherwise the negotiation failed.
